URL of a remote data source (to pull records, apply searches and sorts, submit changes, delete).
String, default = ''
If the .url is defined, then every time the grid needs to refresh the records, they will be pulled from the .url. Every time grid needs to search or sort records, the request will also be submitted to the server to do the search and/or sort.
URL can be defined during object creation: Or any time after the grid has been created: If you do not specify any .url, then the assumed data source is local. You can add/remove records, user can search and sort records and all actions will be performed locally. You can optionally use .load() method to load records from the server without switching grid remote data source.
There is no restriction of setting .url at any given time. Just remember, if you set .url, all searches, sorts, pulling of records (on next page for example), will be loaded from the server.
See grid overview page for data structured passed to and from server side url.
